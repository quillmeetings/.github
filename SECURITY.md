# Security Policy

We take the security of Quill and our users' data seriously. Thank you for
helping keep Quill and its users safe.

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Instead, use one of the following private channels:

- **GitHub private vulnerability reporting** — open a report from the
  **Security** tab of the affected repository (preferred).
- **Email** — [security@quillmeetings.com](mailto:security@quillmeetings.com)

Please include as much of the following as you can:

- The type of issue (e.g. memory corruption, injection, auth bypass, data
  exposure).
- The affected repository, component, and version or commit.
- Step-by-step instructions to reproduce.
- Proof-of-concept or exploit code, if available.
- The impact, including how an attacker might exploit it.

## What to expect

- **Acknowledgement** within 3 business days.
- An assessment and an expected timeline for a fix after triage.
- Progress updates as we work toward a resolution.
- Credit for your report once a fix is released, if you would like it.

## Scope

This policy covers the repositories in the
[`quillmeetings`](https://github.com/quillmeetings) GitHub organization and the
Quill desktop, mobile, and web applications.

Please act in good faith: only test against your own accounts and data, avoid
privacy violations and service disruption, and give us a reasonable opportunity
to resolve an issue before any public disclosure.
