# Contributing

Thanks for your interest in contributing to a Quill project.

These are the organization-wide defaults. Individual repositories may include
their own `CONTRIBUTING.md` with project-specific setup and conventions — when
they do, that file takes precedence.

## Before you start

- Check existing [issues](https://github.com/quillmeetings) and pull requests to
  avoid duplicate work.
- For anything beyond a small fix, open an issue first so we can agree on the
  approach before you invest time.

## Pull requests

- Branch from the repository's default branch.
- Keep changes focused — one logical change per pull request.
- Write a clear description of **what** changed and **why**.
- Make sure existing checks pass before requesting review.
- A maintainer review is required before merge; the default branch is protected.

## Reporting issues

- **Bugs and feature requests** — use the issue templates in the repository.
- **Security issues** — do **not** open a public issue. Follow the
  [security policy](https://github.com/quillmeetings/.github/blob/main/SECURITY.md)
  instead.

## Code of conduct

All participation is governed by our
[Code of Conduct](https://github.com/quillmeetings/.github/blob/main/CODE_OF_CONDUCT.md).
